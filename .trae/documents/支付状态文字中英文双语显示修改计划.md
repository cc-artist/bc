# 支付状态文字中英文双语显示修改计划

## 修改目标
将支付过程中出现的所有文字改为中英文双语显示，保持与其他UI元素一致的设计风格。

## 修改内容

### 1. 正在处理支付消息
**位置**：
- 法相生成窗口：第881行
- 开光结果窗口：第985行

**修改前**：
```
正在处理支付...
```

**修改后**：
```
<span className={styles.englishText}>Processing payment...</span>
<br />
<span className={styles.chineseText}>正在处理支付...</span>
```

### 2. 支付成功消息
**位置**：
- 法相生成窗口：第893行
- 开光结果窗口：第997行

**修改前**：
```
支付成功！
```

**修改后**：
```
<span className={styles.englishText}>Payment successful!</span>
<br />
<span className={styles.chineseText}>支付成功！</span>
```

### 3. 支付失败消息处理
**位置**：第490行（setPaymentError函数调用）

**修改前**：
```
setPaymentError(`支付失败: ${errorMessage}`);
```

**修改后**：
需要修改handlePayment函数，将支付失败消息处理为中英文双语显示。

## 修改步骤

1. **修改法相生成窗口的支付状态消息**：
   - 将第881行的"正在处理支付..."修改为中英文双语
   - 将第893行的"支付成功！"修改为中英文双语

2. **修改开光结果窗口的支付状态消息**：
   - 将第985行的"正在处理支付..."修改为中英文双语
   - 将第997行的"支付成功！"修改为中英文双语

3. **修改支付失败消息处理逻辑**：
   - 修改handlePayment函数，确保支付失败消息也以中英文双语显示

## 设计规范

- 英文文字在上，中文文字在下，使用`<br />`分隔
- 使用现有的`englishText`和`chineseText`CSS类
- 保持与其他UI元素一致的设计风格
- 确保支付错误消息也能正确显示中英文双语

## 预期效果

支付过程中显示的所有文字都将以中英文双语显示，与页面其他元素保持一致，提升国际化用户体验。
