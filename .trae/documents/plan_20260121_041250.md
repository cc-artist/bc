## 现状分析
1. 当前“点亮长明灯”页面使用HTML5 Audio元素播放本地WAV文件，并有Web Audio API作为 fallback
2. 浏览器存在自动播放限制，音频播放需要用户交互触发
3. Bilibili视频有CORS限制，无法直接使用其音频URL
4. 视频URL：`https://www.bilibili.com/video/BV1rBCsB1E42/`

## 解决方案
使用Bilibili embed player iframe，通过以下方式实现自动播放：

1. 添加Bilibili iframe播放器到长明灯容器
2. 隐藏视频画面，只保留音频播放功能
3. 使用Bilibili播放器API控制播放
4. 在用户点击点亮长明灯时触发播放
5. 集成到现有的音频控制按钮

## 具体修改
1. **添加Bilibili播放器iframe**：
   - 在`lightExperienceContainer`中添加iframe元素
   - 设置ref以便编程控制
   - 应用样式使其不可见但仍可播放
   - 配置iframe参数，启用自动播放和隐藏控件

2. **更新音频播放逻辑**：
   - 修改`playAudio()`函数，优先使用Bilibili播放器
   - 在用户点击点亮长明灯时触发iframe播放
   - 处理播放失败的情况

3. **更新音频控制按钮**：
   - 修改点击事件，控制Bilibili播放器的播放/暂停
   - 同步更新按钮状态

4. **移除旧的音频元素**：
   - 移除不再需要的`heartSutraAudioRef`和相关逻辑
   - 移除Web Audio API fallback，因为我们将使用B站播放器

## 预期效果
- 当用户打开长明灯页面并点击点亮长明灯时，自动播放B站视频音频
- 音频控制按钮可以正常控制播放/暂停
- 视频画面不可见，只播放音频
- 符合浏览器自动播放政策，由用户交互触发

## 注意事项
- 需要处理B站播放器的加载延迟
- 确保iframe的src包含正确的自动播放参数
- 考虑移动设备兼容性
- 处理播放器API可能的变化
