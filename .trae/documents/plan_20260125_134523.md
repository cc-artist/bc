# 添加寺庙详情页的支付和咨询按钮

## 问题分析
1. 当前寺庙详情模态框已有两个按钮，但文字和功能不符合要求
2. 需要更新按钮文字为指定内容
3. 需要实现后端相关功能
4. 需要确保按钮功能正常工作

## 解决方案

### 1. 更新 TempleDetailModal 组件
- 修改现有按钮文字为：
  - "支付1万美金订购行程"
  - "咨询客服预约行程"
- 添加新的 props 处理支付功能
- 更新按钮样式和布局

### 2. 扩展组件接口
- 更新 TempleDetailModalProps 接口，添加 onPayment 回调
- 更新组件实现，使用新的回调函数

### 3. 更新父组件（page.tsx）
- 添加 handlePayment 函数处理支付逻辑
- 将支付函数传递给 TempleDetailModal 组件
- 实现支付和咨询的后端相关功能

### 4. 实现后端功能
- 为支付按钮添加模拟支付流程
- 为咨询按钮添加模拟客服预约流程
- 添加适当的状态管理和用户反馈

### 5. 优化用户体验
- 添加加载状态和成功/失败反馈
- 确保按钮功能清晰明确
- 保持与整体设计风格一致

## 技术实现细节
- 使用 React 状态管理支付和咨询流程
- 添加 console.log 模拟后端 API 调用
- 实现简单的状态反馈机制
- 确保类型安全和组件可靠性

## 文件修改
- `src/components/TempleDetailModal.tsx`：主要修改文件
- `src/app/page.tsx`：添加后端功能实现
