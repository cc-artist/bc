# 解决Vercel部署错误：cyber-buddha-blessing: command not found

## 问题分析

### 错误信息
```
sh: line 1: cyber-buddha-blessing: command not found 
Error: Command "cyber-buddha-blessing" exited with 127
```

### 根本原因
1. **项目结构问题**：
   - 根目录包含一个 `package.json` 文件
   - `cyber-buddha-blessing` 目录是实际的Next.js项目，也有自己的 `package.json`
   - Vercel检测到多个 `package.json` 文件，误以为这是一个workspace或monorepo项目

2. **构建命令错误**：
   - Vercel尝试在根目录运行 `cyber-buddha-blessing` 作为命令
   - 但 `cyber-buddha-blessing` 不是可执行命令，而是目录名称
   - 根目录的 `package.json` 没有正确的构建脚本

3. **Vercel自动检测问题**：
   - Vercel自动检测到 `cyber-buddha-blessing` 目录
   - 但错误地将其配置为需要运行的命令，而不是构建目录

## 解决方案

### 方案1：删除根目录的package.json文件（推荐）

**优点**：
- 简单直接，消除Vercel的混淆
- 不影响本地开发流程
- 保持项目结构清晰

**缺点**：
- 需要保留根目录的辅助脚本在其他地方

**步骤**：
1. 删除根目录的 `package.json` 文件
2. 提交更改到GitHub
3. 在Vercel控制台重新部署

### 方案2：在根目录package.json中配置正确的构建命令

**优点**：
- 保留根目录的package.json文件
- 提供明确的构建指令

**缺点**：
- 需要修改配置文件
- 可能需要额外的配置

**步骤**：
1. 修改根目录的 `package.json`，添加构建脚本
2. 配置Vercel使用正确的构建命令
3. 重新部署

### 方案3：在Vercel中明确配置构建设置

**优点**：
- 不需要修改项目文件
- 通过Vercel控制台配置

**缺点**：
- 需要手动配置每个部署

**步骤**：
1. 在Vercel控制台中设置根目录为空（让Vercel自动检测）
2. 明确设置构建命令为 `cd cyber-buddha-blessing && npm run build`
3. 重新部署

## 推荐方案

**推荐使用方案1**：删除根目录的 `package.json` 文件

这是最简单的解决方案，因为：
- 根目录的 `package.json` 只是用于辅助脚本，不是主要项目
- 前端项目是独立的Next.js应用，位于 `cyber-buddha-blessing` 目录
- 删除根目录的 `package.json` 可以消除Vercel的混淆

## 预期结果

- Vercel正确识别 `cyber-buddha-blessing` 目录作为唯一的Node.js项目
- 构建过程正常执行 `npm run build` 命令
- 部署成功完成，网站可以正常访问

## 注意事项

- 删除根目录的 `package.json` 之前，确保没有重要的依赖或脚本
- 辅助脚本（如 `check-assets.js`、`download-background.js` 等）仍然可以本地运行，只是需要手动安装依赖
- 如果需要，可以将这些辅助脚本移动到 `backend` 目录或其他合适的位置
